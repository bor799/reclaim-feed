## 🔐 环境变量配置指南

### 重要原则

**⚠️ 安全第一**：
- ❌ **不要**将真实 API Keys 提交到 Git
- ✅ **使用**平台的环境变量配置面板
- ✅ **或通过**前端设置页面配置（后端保存到 config.yaml）

### 必需环境变量

| 变量名 | 说明 | 示例值 | 配置方式 |
|--------|------|--------|----------|
| `ZHIPU_API_KEY` | 智谱 AI 密钥 | `your_actual_key_here` | 平台面板或前端界面 |
| `TELEGRAM_BOT_TOKEN` | Telegram 机器人 | `bot123456:ABC-DEF` | 前端界面配置 |
| `TELEGRAM_CHAT_ID` | Telegram 聊天 ID | `123456789` | 前端界面配置 |

### 可选环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `TZ` | 时区 | `Asia/Shanghai` |
| `AUTH_ENABLED` | 是否启用鉴权 | `False` (测试阶段) |
| `PORT` | 服务端口 | `8000` (默认) |

## 🛠️ 故障排查

### 问题 1：部署后无法访问

**可能原因**：
- 端口未正确暴露
- 防火墙阻止
- 健康检查失败

**解决方案**：
```bash
# 检查服务日志
# Railway: 查看部署日志
# AI Builder: 使用 GET /v1/deployments/{service_name}/logs
```

### 问题 2：API 调用失败

**可能原因**：
- 环境变量未配置
- API Key 无效
- 网络问题

**解决方案**：
1. 在前端设置页面重新配置
2. 使用 "Test Connection" 按钮测试
3. 检查后端日志

### 问题 3：配置丢失

**可能原因**：
- 使用了临时容器
- 数据未持久化

**解决方案**：
- Railway：自动持久化 `config/` 目录
- 其他平台：配置 Volume 映射
